Dans cet article, je vais expliquer pas à pas comment j'ai implémenté le système de blog de ce portfolio. L'objectif : un blog simple, performant, avec du contenu en MDX et une validation stricte des métadonnées.

## Architecture globale

Le blog repose sur 4 piliers :

1. **Fichiers MDX** : contenu stocké dans `src/content/`
2. **Utilitaires serveur** : parsing et validation dans `lib/blog-utils.ts`
3. **Routes dynamiques** : pages Next.js dans `app/blog/` et `app/blog/[slug]/`
4. **Composant MDX** : rendu du contenu avec `next-mdx-remote-client`

## Structure des dossiers

```
src/
├── app/
│   └── blog/
│       ├── page.tsx           # Liste des articles
│       └── [slug]/
│           └── page.tsx       # Article individuel
├── content/

├── features/
│   └── mdx/
│       └── mdx.tsx            # Composant de rendu MDX
└── lib/

```

## Étape 2 : Créer les utilitaires serveur

Créez `src/lib/blog-utils.ts` :

```typescript
import fs from 'fs/promises';
import path from 'path';
import z from 'zod';
import matter from 'gray-matter';

// Chemin vers les articles
const blogPostDirectory = path.join(process.cwd(), 'src', 'content');

// Schéma de validation Zod
const postSchema = z.object({
  title: z.string().min(45).max(60),
  description: z.string().optional(),
  date: z.coerce.string(),
  lastUpdated: z.string().optional(),
  tags: z.array(z.string()).optional(),
  readingTime: z.string().optional(),
});

type BlogPost = z.infer<typeof postSchema> & {
  slug: string;
  content: string;
};

// Récupérer tous les articles
export const getBlogPosts = async () => {
  const files = await fs.readdir(blogPostDirectory);
  const fileNames = files.filter(file => file.endsWith('.mdx'));
  const blogPosts: BlogPost[] = [];

  for await (const fileName of fileNames) {
    const fullPath = path.join(blogPostDirectory, fileName);
    const fileContents = await fs.readFile(fullPath, 'utf8');
    const frontMatter = matter(fileContents);

    // Validation Zod
    const safeData = postSchema.safeParse(frontMatter.data);
    if (!safeData.success) {
      console.error(`❌ Validation failed for ${fileName}`);
      continue;
    }

    blogPosts.push({
      ...safeData.data,
      slug: fileName.replace(/\.mdx?$/, ''),
      content: frontMatter.content,
    });
  }

  return blogPosts;
};

// Récupérer un article par slug
export const getBlogPost = async (slug: string) => {
  const blogPosts = await getBlogPosts();
  return blogPosts.find(post => post.slug === slug);
};
```

### Points clés

- **Validation stricte** : le titre doit faire entre 45 et 60 caractères (SEO)
- **Slug automatique** : généré à partir du nom de fichier
- **Type-safe** : TypeScript infère le type depuis le schéma Zod
- **Gestion des erreurs** : les articles invalides sont ignorés silencieusement

## Étape 3 : Page d'index du blog

Créez `src/app/blog/page.tsx` :

```tsx
import { Card, CardDescription, CardFooter, CardHeader, CardTitle } from "@/components/ui/card";
import { getBlogPosts } from "@/lib/blog-utils";
import Link from "next/link";

export default async function BlogIndex() {
  const posts = await getBlogPosts();

  return (
    <main style={{ maxWidth: 760, margin: "40px auto", padding: "0 16px" }}>
      <h1>Blog</h1>
      <ul style={{ paddingLeft: 16 }}>
        {posts.map((post) => (
          <li key={post.slug}>
            <Card>
              <CardHeader>
                <p>{new Date(post.date).toLocaleDateString('fr-FR')}</p>
                <CardTitle>{post.title}</CardTitle>
                <CardDescription>{post.description}</CardDescription>
              </CardHeader>
              <CardFooter>
                <Link href={`/blog/${post.slug}`}>
                  Lire l'article →
                </Link>
              </CardFooter>
            </Card>
          </li>
        ))}
      </ul>
    </main>
  );
}
```

### Ce qui se passe ici

- **Server Component** : `getBlogPosts()` s'exécute côté serveur
- **Composants shadcn/ui** : pour un design cohérent
- **Tri automatique** : on peut ajouter `.sort()` par date si besoin

## Étape 4 : Page article dynamique

Créez `src/app/blog/[slug]/page.tsx` :

```tsx
import { notFound } from "next/navigation";
import { getBlogPost } from "@/lib/blog-utils";
import { Mdx } from "@/features/mdx/mdx";

export default async function BlogPost(props: { 
  params: Promise<{ slug: string }> 
}) {
  const params = await props.params;
  const blogPost = await getBlogPost(params.slug);

  if (!blogPost) {
    notFound();
  }

  return (
    <main className="w-full max-w-3xl mx-auto my-10 px-4">
      <article className="max-w-none">
        <Mdx>{blogPost.content}</Mdx>
      </article>
    </main>
  );
}
```

### Explications

- **Route dynamique** : `[slug]` capture le nom de l'article
- **notFound()** : retourne une 404 si l'article n'existe pas

## Étape 5 : Composant de rendu MDX

Créez `src/features/mdx/mdx.tsx` :

```tsx
import { MDXRemote } from "next-mdx-remote-client/rsc";
import { Suspense } from "react";
import remarkGfm from "remark-gfm";

export const Mdx = ({ children }: { children: string }) => {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <MDXRemote 
        source={children}
        options={{
          mdxOptions: {
            remarkPlugins: [remarkGfm],
          },
        }}
      />
    </Suspense>
  );
};
```

### Pourquoi ce composant ?

- **next-mdx-remote-client** : compile le MDX en React Server Component
- **remark-gfm** : active les tables, les listes de tâches, etc.
- **Suspense** : améliore la performance avec le streaming SSR

### 5. Métadonnées SEO dynamiques

Ajoutez dans `app/blog/[slug]/page.tsx` :

```tsx
export async function generateMetadata(props: { 
  params: Promise<{ slug: string }> 
}) {
  const params = await props.params;
  const post = await getBlogPost(params.slug);
  
  if (!post) return {};

  return {
    title: post.title,
    description: post.description,
    openGraph: {
      title: post.title,
      description: post.description,
      type: 'article',
      publishedTime: post.date,
    },
  };
}
```
